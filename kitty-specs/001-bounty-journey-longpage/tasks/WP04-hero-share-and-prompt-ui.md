# WP04 - Hero Share and Prompt UI

## Objective

完成页面中最“传播友好”的三块内容：Hero、分享结果区、AI Agent 指令区。这个 WP 决定用户是否能一眼理解页面是什么，以及是否愿意截图/复制继续传播。

## Context

- Share 区的对外主语必须是 `Agent`。
- Agent Prompt 区必须适配当前项目和 skill 仓库，不得写成泛化文案。
- Hero/Share/Prompt 的内容应该与 Journey 区形成前后呼应，但不依赖 Journey 才能成立。

## Subtask Guidance

### T016 - Build HeroSection with primary narrative and CTA

**Purpose**

Create the top-of-page entry point that tells users what this page is and why they should scroll.

**Implementation Guidance**

1. Render eyebrow, headline, summary and CTA from content data.
2. Position the Hero so it can carry the page even before the user reaches Journey.
3. The CTA should invite the user into the main simulation flow.
4. Keep the Hero visually bold, not generic marketing boilerplate.

**Validation**

- [ ] Hero explains the page in one scan
- [ ] CTA clearly points toward the journey section

### T017 - Build share summary card and supporting stat components

**Purpose**

Turn the “public result state” into a share-ready UI block that highlights score, resonance and faction status.

**Implementation Guidance**

1. Create a primary share card component and any stat pill/row helpers needed.
2. Show at minimum:
   - `Agent 打分多少`
   - `已共振`
   - `已加入阵营`
3. Add limited supporting details such as token or Telegram follow-up only if they reinforce the main result.
4. Keep the card screenshot-friendly and easy to scan.

**Validation**

- [ ] The three core result signals are immediately visible
- [ ] Share wording uses `Agent`, not `龙虾打分`
- [ ] The card reads well even outside the full page context

### T018 - Build Agent Prompt section with copy interaction

**Purpose**

Give users a concrete handoff block they can copy into an AI Agent to continue the real flow.

**Implementation Guidance**

1. Render one or more prompt cards from content data.
2. Include:
   - prompt title
   - goal
   - context
   - expected output
   - copy action
3. Copy behavior should remain local and simple; do not attempt external execution.
4. Make sure the prompt body explicitly references the Claws Temple skill repo.

**Validation**

- [ ] At least one prompt card is copyable
- [ ] Prompt content is clearly project-specific
- [ ] Copy interaction is keyboard accessible

### T019 - Add shared UI atoms for non-journey sections

**Purpose**

Avoid duplicated markup across Hero, Share and Prompt sections while keeping the abstraction level modest.

**Implementation Guidance**

1. Build only the shared atoms actually needed, such as section headings or copy buttons.
2. Keep the abstractions light; do not build a generic design system.
3. Ensure these atoms support the bold visual language already planned.

**Validation**

- [ ] Shared atoms remove duplication without obscuring intent
- [ ] No unnecessary abstraction layer is introduced

### T020 - Validate Agent-first wording and forbidden visible terms

**Purpose**

Protect the user-confirmed language contract and the Claws Temple public vocabulary.

**Implementation Guidance**

1. Audit Hero, Share and Prompt rendering for:
   - `Agent` as the visible subject where required
   - no `龙虾打分多少` in the share result
   - no visible `ca_hash`, `钱包`, or similar forbidden terms
2. Fix copy inside owned files if any term drifts.
3. Keep the final copy aligned with the feature spec and brand lexicon.

**Validation**

- [ ] Share card is Agent-first
- [ ] Prompt section remains project-specific
- [ ] Forbidden visible terms do not appear in owned files

## Definition of Done

- Hero establishes the page quickly
- Share card is screenshot-ready and semantically correct
- Agent Prompt cards are copyable and project-specific
- Non-journey sections obey the public wording contract

## Risks

- If the share card overexplains, the propagation value drops
- If the prompt section is too generic, it fails the main use case

## Reviewer Guidance

- Read the Share block as if it were a standalone image caption
- Verify the Prompt block gives a real next step, not abstract advice
- Verify the user-approved wording change to `Agent` is preserved
