# WP02 - Clawvard Challenge Surface Refit

## Objective

把首屏、分享区和 Agent Prompt 改成真正能拿出去发的挑战面：视觉结构优先参考本地 Clawvard 下载页，用户一上来先看到分数卡、挑战文案和 CTA，而不是 oversized 标题和内部说明。

## Context

- 首屏第一焦点应该是成绩卡，而不是大标题。
- 要删除“首页先只看……”和“演示页，不代替真实注册……”这类内部 copy。
- 首页 challenge 文案要直接表达：分数、等级、阵营、已共振、已加入阵营。
- Share 区不是摘要组件，而是“可截图 / 可复制 / 可转发”的战书。
- Agent Prompt 区仍保留 terminal 风格，但应该在首屏 CTA 之后紧接出现。
- 视觉层次优先向参考页靠拢，不需要硬套上一轮的色彩方向。

## Subtask Guidance

### T005 - Refit the hero card around the Clawvard layout

**Purpose**

Make the first screen visually read like a score challenge card instead of a landing-page hero.

**Implementation Guidance**

1. Follow the downloaded reference page's hierarchy:
   - light top chrome
   - single white score card
   - thin gradient accent bar
   - grade tile + large numeric score
   - warm challenge blurb
   - strong stacked CTAs
2. Reduce or demote the oversized heading so the result card becomes the focal point.
3. Preserve responsiveness for narrow mobile widths without horizontal scroll.
4. Keep the component structure intentional and readable; do not dump everything into one mega component.

**Validation**

- [ ] The score card dominates the first viewport
- [ ] The title no longer overpowers the result surface
- [ ] The layout still works on mobile widths

### T006 - Replace internal-facing copy with public challenge copy

**Purpose**

Ensure every visible sentence on the public surface sounds like a challenge invitation, not an internal product note.

**Implementation Guidance**

1. Remove internal-only explanation strings from user-visible surfaces.
2. Write localized challenge copy that directly includes:
   - seeded score
   - grade
   - faction
   - resonance completion
   - joined-faction state
3. Keep the tone public and provocative, not implementation-explanatory.
4. Make sure both Chinese and English versions stay semantically aligned.

**Validation**

- [ ] No internal-facing explanatory sentence remains visible
- [ ] Hero challenge copy surfaces score / grade / faction / resonance / joined state
- [ ] The public subject stays `Agent`

### T007 - Rebuild the share section as a two-mode challenge card

**Purpose**

Turn the share block into a proper “forward this challenge” surface rather than a stats panel.

**Implementation Guidance**

1. Keep the `share image` and `share text` tabs.
2. Make the visual mode feel card-like and reference-adjacent:
   - centered score result
   - clear challenge framing
   - concise supporting lines
3. Make the text mode a forwardable challenge letter with copy affordance.
4. Keep seeded result fields consistent with Hero and Journey.

**Validation**

- [ ] Image and text modes both feel like shareable challenge surfaces
- [ ] The copied text contains score / grade / faction / resonance context
- [ ] The section still supports bilingual copy

### T008 - Tighten typography, tokens, and prompt-card hierarchy

**Purpose**

Give the public surfaces a cleaner editorial rhythm so the page feels deliberate rather than improvised.

**Implementation Guidance**

1. Update the visual token layer only as needed for this new reference direction.
2. Tighten type scale and spacing to avoid display-heading excess.
3. Keep the Agent Prompt block immediately after the Hero CTA cluster.
4. Preserve the terminal-card affordance and copy action, but tune it to match the new score-card hierarchy.

**Validation**

- [ ] Type scale is calmer and more reference-aligned
- [ ] Prompt card sits in the right narrative position
- [ ] Tokens and spacing feel consistent across Hero, Share, and Prompt

## Definition of Done

- Hero visually centers on the score challenge card.
- Internal-only copy has been removed from public surfaces.
- Share image/text modes read like challenge assets.
- Prompt card fits the new public hierarchy and remains usable.

## Risks

- If the refit only copies cosmetics without changing information hierarchy, the page will still read as an internal demo.
- If Chinese and English copy drift apart, the public challenge contract will become inconsistent across locales.

## Reviewer Guidance

- Compare the first-screen structure against the local Clawvard download, not against the previous site version.
- Check that the score card is the first focal point.
- Check that no internal QA-style copy leaks into the public surface.
