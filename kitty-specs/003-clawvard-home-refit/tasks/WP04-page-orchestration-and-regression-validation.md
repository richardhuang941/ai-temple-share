## Subtask Guidance

### T013 - Wire CTA anchors and section orchestration

**Purpose**

Make the user flow feel intentional: accept challenge goes to prompt, share challenge goes to share, watch simulation goes to Journey and starts it.

**Implementation Guidance**

1. Confirm section order supports the narrative:
   - Hero
   - Agent Prompt
   - Share
   - Journey
2. Wire or refine anchor IDs and scroll targets for the three main CTAs.
3. Ensure the Journey-start CTA can coordinate with the downstream playback start behavior.
4. Keep the orchestration layer thin and readable.

**Validation**

- [ ] Accept challenge lands on the prompt section
- [ ] Share challenge lands on the share section
- [ ] Watch simulation lands on Journey and triggers playback

### T014 - Ensure locale and seeded-result consistency

**Purpose**

Prevent last-mile integration from drifting the copy or result data between sections and locales.

**Implementation Guidance**

1. Verify top-level bundle assembly includes the seeded result contract from WP01.
2. Keep system-locale default and stored override behavior intact.
3. Ensure all sections consume the same localized result values.
4. Patch any export or composition gaps needed to keep the contract coherent.

**Validation**

- [ ] Locale default behavior still works
- [ ] Manual locale switching still works
- [ ] Score / grade / faction / resonance copy stays aligned across sections

### T015 - Update integration coverage

**Purpose**

Lock the public page behaviors that matter most in this refit: bilingual switching, CTA routing, and current-task visibility.

**Implementation Guidance**

1. Expand integration tests to cover:
   - default locale selection
   - manual locale toggle
   - CTA anchor behavior
   - Journey trigger behavior
   - key seeded-result copy appearing consistently
2. Add assertions that are resilient to layout refactors.
3. Keep tests centered on user-visible behavior.

**Validation**

- [ ] The integration suite would catch broken CTA routing
- [ ] The integration suite would catch broken locale switching
- [ ] The integration suite would catch obvious seeded-result drift

### T016 - Run build/test/preview validation

**Purpose**

Finish the mission with evidence, not with “looks okay on my machine”.

**Implementation Guidance**

1. Run the relevant tests and build.
2. Smoke-test the preview flow.
3. Fix the highest-signal acceptance issues discovered during verification.
4. Leave the branch in a clean, reviewable state.

**Validation**

- [ ] `npm run test` passes
- [ ] `npm run build` passes
- [ ] The preview is usable for manual review

## Definition of Done

- CTA routing and section order are coherent.
- Locale behavior remains stable.
- Integration coverage reflects the new public challenge flow.
- Test/build verification succeeds.

## Risks

- If App-level orchestration is too implicit, sections may individually look correct while the user flow still feels broken.
- If tests only assert legacy copy, they will miss the actual regressions introduced by this refit.

## Reviewer Guidance

- Review the app flow from top to bottom, not just individual sections.
- Verify CTA routing and locale switching from the user's perspective.
- Verify the seeded result contract survives the final composition layer.
